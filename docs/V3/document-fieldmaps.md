# Document Field Maps — ทุกเอกสารในลูป (GL-R-ERP)

> อ่านคู่กับ `change-requests-v3.md` (ข้อ 12, 13.1, 13.2, 13.5)
> วิธี generate ทุกเอกสาร = **เปิด template จริงด้วย POI แล้วเติมเฉพาะ dynamic cells** (ห้ามวาดใหม่ — ดูข้อ 12)
> template .xlsx สะอาดอยู่ที่ `docs/templates/` : quotation_template / deposit_notice_template / remaining_invoice_template

---

## 0. โครงร่วมของ 3 เอกสาร Excel (ใบเสนอราคา / ใบมัดจำ / ใบแจ้งหนี้ส่วนที่เหลือ)
เลย์เอาต์ฐานเดียวกันหมด ต่างกันแค่ **หัวเอกสาร + prefix เลข + บล็อกสรุปยอด (VAT)**
- หัวบริษัท (B1–B5) = คงที่
- หัวตาราง (แถว 10): `ลำดับ | รายละเอียด | จำนวน | หน่วย | ราคา | ส่วนลด | คงเหลือ | เป็นเงิน(บาท)`
- **สูตรต่อบรรทัดสินค้า** (มีอยู่ใน template แล้ว — แค่ใส่ ราคา(E)/ส่วนลด(F,G)/จำนวน(C)):
  - `คงเหลือ (H)` = ถ้า `G="พิเศษ"` → ใช้ราคาพิเศษ (คอลัมน์ **K** ในใบเสนอราคา / **J** ในใบมัดจำ) ; ถ้า `G="Net"`/ไม่มีส่วนลด → = ราคา(E) ; ปกติ = `E − E×F` (F = ส่วนลด **%**)
  - `เป็นเงิน (I)` = `H × C`
- หมายเหตุ / ลายเซ็น / form control = ปล่อยตาม template
- **เลขเอกสาร** = prefix + ปีไทย 2 หลัก + running (แยกซีรีส์ต่อ docType)
- **ชื่อผู้เซ็น**: ในไฟล์ทำ lookup จากรหัส salesperson (ช่อง `I3`) → คอลัมน์ M → **ในแอปใช้ตาราง `signatories` (code→name) แทน**
- **ที่อยู่ลูกค้า**: ในไฟล์ใช้ `VLOOKUP(B7,Sheet1!...)` (customer master) → **ในแอปดึงจากตาราง `customers` แทน** (ไฟล์ remaining เป็น `#REF!` อยู่ อย่าพึ่ง VLOOKUP)

## 1. ใบเสนอราคา — docType `QUOTATION` — prefix `QN` — form `F-SM-002(03)`
| ช่อง | เซลล์ | ที่มา |
|---|---|---|
| หัว "ใบเสนอราคา" | H1 | คงที่ |
| วันที่ | B4 | วันที่ออก (template = TODAY) |
| เลขที่อ้างอิง | I4 | `QN`+ปีไทย+running |
| ฝ่าย / รหัส Sales | H3 / I3 | I3 = รหัส salesperson (คุมชื่อผู้เซ็น A45) |
| หน่วยงาน | H5 | จากงาน |
| Sales (ชื่อ+เบอร์) | H6 | ผู้ใช้ Sales |
| เรียน (ผู้ติดต่อ+บริษัท+เลขภาษี) | A5/B5 | customers+contacts |
| โทร/อีเมลลูกค้า | B6 | contacts |
| Project | B8 | projects |
| รายการสินค้า | แถว ~8–37 | ticket_items ที่อนุมัติ (แต่ละรายการกินหลายแถวได้: แถวหลัก + sub-row หมายเหตุจำนวน/บรรจุ) |
| หมายเหตุ ข้อ 1,2,3,7 (แก้ได้) | B24/B26/B28/B35 | offerDate / depositPercent / importLeadDays / priceValidityDays |
| **สรุปยอด** | I38/I39/I40 | `รวม=SUM(items)` ; `VAT=รวม×7%` (**บนยอดเต็ม**) ; `ทั้งสิ้น=รวม+VAT` |

## 2. ใบแจ้งยอดเงินรับมัดจำ — docType `DEPOSIT_NOTICE` — prefix `GLRD`
| ช่อง | เซลล์ | ที่มา |
|---|---|---|
| หัว "ใบแจ้งยอด / เงินรับมัดจำ" | H2/H3 | คงที่ |
| วันที่ / เลขที่ / อ้างอิง | I7 / H8 / H9 | ออก / `GLRD`+ปีไทย+run / อ้างใบเสนอราคา-PO |
| เรียน / ที่อยู่ | B7 / B8 | customers (ที่อยู่จากตาราง customers) |
| Project | B11 | projects |
| รายการสินค้า | แถว ~13–43 | items (special price = คอลัมน์ **J**) |
| **สรุปยอด** | I44/I45/I46/I47 | `รวม=ROUND(SUM,2)` ; `มัดจำ=ROUND(รวม×depositPct,2)` ; `VAT=ROUND(มัดจำ×7%,2)` (**บนมัดจำ**) ; `ต้องชำระ=มัดจำ+VAT` |

## 3. ใบแจ้งหนี้ส่วนที่เหลือ — docType `REMAINING_INVOICE` — prefix `GLR`
| ช่อง | เซลล์ | ที่มา |
|---|---|---|
| หัว "ใบแจ้งยอด" | H2 | คงที่ |
| วันที่ / เลขที่ / อ้างอิง | H7 / H8 / H9 | ออก / `GLR`+ปีไทย+run / อ้างใบเสนอราคา-PO |
| เรียน / ที่อยู่ / Project | B7 / B8 / B11 | customers / projects |
| รายการสินค้า (ยอดเต็ม) | แถว ~13+ | items (ยอดเต็มของสินค้า) |
| **บรรทัดหักมัดจำ** | เช่น แถว15 | `B="หัก มัดจำ {ref}"`, `I = −depositAmount` (บรรทัดลบ) |
| **สรุปยอด** | I42/I43/I44 | `รวม=SUM(items)−มัดจำ` (=คงเหลือ) ; `VAT=คงเหลือ×7%` (**บนคงเหลือ**) ; `ต้องชำระ=คงเหลือ+VAT` |
> ตัวอย่างตรวจ: 37,114.18 − 18,557.09 = 18,557.09 → VAT 1,298.9963 → **19,856.0863** ✓

## 4. Import Request — docType `IMPORT_REQUEST` — prefix `IR` — form `F-SM-001(04)` — **template เป็น PDF**
- ⚠ template ให้มาเป็น **PDF (ไม่มี Excel)** → แนะนำ **สร้าง .xlsx เลียนแบบ layout** เพื่อใช้ pipeline เดียวกัน (POI เติม → LibreOffice→PDF) ; ใช้ PDF ที่ให้มาเป็นภาพอ้างอิง
- **1 โรงงาน = 1 ใบ** (Brand = ชื่อโรงงาน) → group `ticket_items` ตาม `factory` (จากฐาน catalog) แล้วออกใบแยก
- หัว: `Brand`(โรงงาน), `ReF No` = `IR`+ปีไทย+running (เช่น IR69068), `Request date`
- ตาราง: `Item | Code | Size | จำนวน | หน่วยนับ | สั่งมาให้(=ลูกค้า) | Requested by | กำหนดวันที่ต้องการของ`
- มีแถว `Project : ...` และหมายเหตุ `สั่งตามPO`
- ท้าย: กำหนดเรือเข้าโดยประมาณ, Checked date/By (Buyer), Approve date/By (GM/MD), Request By, วันที่ได้รับมัดจำ, ลงชื่อ, ผู้อนุมัติ
- สถานะที่เกี่ยว (จาก change-requests ข้อ 13.2): `IR_ISSUED`(ออกใบ IR) → `IR_SENT`(ส่ง IR ให้โรงงาน)

## 5. สรุปฐาน VAT ต่างกันของแต่ละเอกสาร (จุดพลาดง่าย — ต้องใช้ template สรุปยอดคนละแบบ)
| เอกสาร | ฐาน VAT 7% | ยอดที่ต้องชำระ |
|---|---|---|
| ใบเสนอราคา | ยอดรวมเต็ม | รวม + VAT |
| ใบมัดจำ | ยอดมัดจำ | มัดจำ + VAT(มัดจำ) |
| ใบแจ้งหนี้ส่วนที่เหลือ | ยอดคงเหลือ (รวม−มัดจำ) | คงเหลือ + VAT(คงเหลือ) |
> ผลรวม VAT ทั้งดีล = VAT ของยอดเต็ม (มัดจำ + คงเหลือ) — ถูกต้องตามภาษีไทย (มัดจำถึง tax point ก่อน)

## 6. mapping ข้อมูล → เอกสาร (แหล่งข้อมูลร่วม)
- หัวลูกค้า: `customers`(name,taxId,address) + `contacts` + `projects`
- รายการ: `document_items` (snapshot จาก ticket_items ที่อนุมัติ) = รายละเอียด/จำนวน/หน่วย/ราคา/ส่วนลด(%,หรือ "พิเศษ"→ราคาพิเศษ)/คงเหลือ/เป็นเงิน
- เลขรัน: sequence แยกต่อ prefix (QN / GLRD / GLR / IR) + ปีไทย
- ผู้เซ็น: `signatories` (รหัส→ชื่อ)
- depositAmount (ใช้ในใบแจ้งหนี้ส่วนที่เหลือ) = มัดจำที่ออกไปในใบมัดจำ (ผูกกับ document ก่อนหน้าใน ticket เดียวกัน)

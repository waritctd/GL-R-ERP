Cross-cutting notes

  • Undefined permission: canViewSensitiveEmployeeData is not in ROLE_PERMISSIONS → salary/sensitive tab hidden for everyone (UX-09).
  • Confirm-dialog inconsistency: most destructive actions use ConfirmDialog; ProfileRequests has none; PriceImport uses native window.confirm.
  • Validation split: HR/leave/OT/employee/password use react-hook-form + zod with accessible inline errors; the sales stack uses imperative validation + toast/one-line strings (UX-03).
  • Route-guard gaps: /ceo-settings, /catalog, /attendance sit outside RequireAccess (nav-gated only).

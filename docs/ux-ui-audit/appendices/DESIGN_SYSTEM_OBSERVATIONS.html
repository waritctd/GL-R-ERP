Tokens (index.css @theme static)

Color

Brand --color-primary #4f46e5/-hover #6366f1, accent #14b8a6. Semantic sets for success/warning/danger/info each with fg/bg/border. Deal-pipeline phase accents --color-phase-1..5. ~90 flat color vars; neutrals + surfaces + a text ramp (ink → faint). Sidebar #0b1220. Light only (no dark scheme).

Type

Sarabun / system; sizes 11–34px (px, not rem); weights hardcoded inline (no weight tokens).

Spacing / radius / shadow

4px base, --space-1..8; radius sm3/md8/lg20/pill999; shadows sm/md/lg + focus-ring + inset-active.

Breakpoints

No tokens — hardcoded literals (720 mobile, 721–1040 icon-rail, 1040, 900 ticket-detail). useIsMobile mirrors 720px.

Legacy CSS (styles.css ≈ 2,100 lines, @layer legacy)

Still styles: reset/base (global input min-height 40px, label grid), auth, app-shell/sidebar/topbar, a full parallel button system (.primary-button…), cards/panels, grid "tables", status badges, modals, toasts, skeletons, ~400 lines of media queries with table→card reflow.

Dead/duplicated: a full duplicate :root token block (shadowed by @theme); unused .tabs/.status-tabs/.status-tab (~75 lines); button styling duplicated vs Button.jsx (44px floor patched twice). See UX-11, UX-29.

Inconsistencies (evidence in code)

  • Two button systems — cva Button vs legacy classes across 9 sales files (UX-11).
  • Two tone vocabularies — StatCard color-names vs StatusBadge semantic-names; same green aliased 3 ways (UX-17).
  • Per-feature status→tone maps — no shared helper (UX-17).
  • Inline hex instead of tokens — TicketDetail ~264, TicketCreate ~111, Deposit ~77 inline styles (UX-18).
  • Modal-primitive drift — CeoSettings bespoke overlay; PriceImport native confirm (UX-07/08).
  • Empty-state rendered two ways — EmptyState component vs raw div in dashboard.

Accessibility-in-code & color-as-status

Focus traps (Modal + AppShell drawer), aria-live toasts, aria-hidden icons, dl/dt semantics, breadcrumb nav — all present. Gaps: FormField doesn't auto-inject aria-invalid/aria-describedby (UX-27); collapsible titles aren't headings (UX-25). Status color is generally backed by a text label; the exceptions are timeline event dots and the current-step left border, which are color-only (UX-16).

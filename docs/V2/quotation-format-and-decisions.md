## 2. ตัวเลขที่ Sales แก้ได้ (ฝังในหมายเหตุข้อ 1,2,3,7)
ทำเป็น **field พารามิเตอร์** แล้ว compose ข้อความหมายเหตุจาก template (ไม่ให้ Sales พิมพ์ทับทั้งประโยค):
| ข้อ | field | ตัวอย่างในไฟล์ | ข้อความ template |
|---|---|---|---|
| 1 | `offerDate` (วันที่ได้รับจำนวน/วันที่แก้ไข) | 19/05/2569 | "จำนวนที่เสนอ...ได้รับมาเมื่อวันที่ {offerDate}" |
| 2 | `depositPercent` (% มัดจำ) | 50% | "บริษัทฯ ขอรับมัดจำ {depositPercent}% เมื่อสั่งซื้อ..." |
| 3 | `importLeadDays` (ระยะเวลานำเข้า) | 90 วัน | "...ระยะเวลานำเข้าประมาณ {importLeadDays} วัน" |
| 7 | `priceValidityDays` (ยืนราคา) | 30 วัน | "กำหนดยืนยันราคา : {priceValidityDays} วัน นับจากวันที่..." |
> ข้อ 4,5,6,8 = คงที่ (boilerplate) ; เก็บทั้งชุดใน `document_note_templates` (docType=QUOTATION)

## 3. สรุปยอด — ⚠️ ต่างจากใบแจ้งมัดจำ
```
รวมเป็นเงิน (I38)        = SUM(รายการ)                 = 129,280.8
ภาษีมูลค่าเพิ่ม (I39)     = รวมเป็นเงิน × 7%  ← บน "ยอดรวมเต็ม"  = 9,049.656   [H39=0.07]
รวมเป็นเงินทั้งสิ้น (I40)  = รวมเป็นเงิน + VAT            = 138,330.456
```
> ⚠️ **VAT ของใบเสนอราคาคิดจากยอดรวมเต็ม** (ต่างจากใบแจ้งมัดจำที่คิดจากยอดมัดจำ)
> มัดจำในใบเสนอราคาเป็นแค่ "เงื่อนไขในหมายเหตุ" ไม่เอามาคิดในยอดสุทธิ
> → docType QUOTATION กับ DEPOSIT_NOTICE ใช้ **template สรุปยอดคนละแบบ** (engine generic รองรับอยู่แล้ว)

## 4. เลขที่เอกสาร — แยกชุดต่อ docType
- QUOTATION = `QN`+ปีไทย+ลำดับ ; DEPOSIT_NOTICE = `GLRD`+ปีไทย+ลำดับ → **เลขรันแยกซีรีส์ต่อ docType**

## 5. ผู้เซ็น (บล็อกขวา K/L/M ในไฟล์ = lookup ภายใน)
- คอลัมน์ L/M เก็บ code→ชื่อผู้เซ็น (เช่น จินตนา หาญมนตรี, ราม อิฐรัตน์ …) เป็น **ตาราง lookup** ไม่ใช่ส่วนพิมพ์
- ทำเป็นตาราง `signatories` (`code`,`role`,`name`) → เติมชื่อในบล็อกลายเซ็น (แถว44–46) ตาม Sales/ผจก.ที่เกี่ยวข้อง

## 6. ปิดคำถาม R3/R4 (จากที่ผู้ใช้ตอบ)

### R3 — brand → factory: ดึงจากฐาน catalog
- ใช้ field **`factory`** ในตาราง `products` (จาก `catalog_extract.py`) เป็นตัว map ยี่ห้อ/สินค้า → โรงงาน
- Import Request จัดกลุ่มรายการตาม `factory` แล้วแยกอีเมล/เอกสารต่อโรงงาน (ไม่ต้องตั้ง mapping มือ ถ้า catalog มี factory ครบ)
- ✅ acceptance: เลือกสินค้าจาก catalog → ระบบรู้โรงงานเอง → แยก Import Request อัตโนมัติ

### R4 — FX rate: กรอกเอง/อ้าง BOT + base config + custom ต่อดีล
- **FX**: CEO กรอก **รายเดือน** เก็บใน `fx_rates` (`currency`,`rate`,`month`,`source`,`updatedBy`,`updatedAt`)
  - อ้างอิงอัตราจาก BOT: https://www.bot.or.th/en/statistics/exchange-rate.html (เก็บ source ไว้)
  - (อนาคต) เผื่อ hook ดึง BOT อัตโนมัติภายหลัง — ตอนนี้กรอกมือพอ
- **Price calc = base + override ต่อดีล**:
  - `price_calc_config` เก็บ **base** (ชุด factor มาตรฐาน + version)
  - แต่ละ ticket/quotation **override ค่า factor เฉพาะดีลได้** (เก็บ `calcOverrides` ต่องาน) โดยไม่แก้ base
  - ราคาที่คำนวณเก็บ `calcConfigVersion` + override ที่ใช้ → ตรวจย้อนได้ว่าดีลนี้คิดด้วยอะไร
- ✅ acceptance: CEO ตั้ง base ได้; ในดีลหนึ่งปรับ freight/margin เฉพาะดีลได้โดย base ไม่เปลี่ยน; ราคาคำนวณอัปเดตตาม

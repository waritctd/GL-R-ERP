# FIX: Document Generation ต้องใช้ไฟล์ Template จริง (ห้ามสร้างเอง)

> **ปัญหาปัจจุบัน**: ระบบ generate เอกสารออกมา "ไม่ตรง template" เพราะโค้ดสร้าง workbook/เลย์เอาต์ขึ้นใหม่เอง
> **หลักการแก้ (กฎเหล็ก)**: เปิดไฟล์ template จริงด้วย POI → เติมค่าเฉพาะเซลล์ที่ระบุในเอกสารนี้ → เซฟ/แปลง PDF
> ทุกอย่างนอกเหนือจากเซลล์ที่ระบุ **ห้ามแตะเด็ดขาด** (โลโก้ เส้นตาราง merged cells ความกว้างคอลัมน์ ลายเซ็น form control)
>
> ไฟล์ template อยู่ที่ `docs/templates/` (sheet ชื่อ `Update` ทุกไฟล์):
> - `Template_ใบเสนอราคา.xls` → QUOTATION
> - `Template_ฟอร_มใบแจ_งร_บเง_นม_ดจำ.xls` → DEPOSIT_NOTICE
> - `Template_ฟอร_มใบแจ_งหน__ส_วนท__เหล_อ.xls` → REMAINING_INVOICE
> - `Template_Import_Request.pdf` → IMPORT_REQUEST (สร้าง .xlsx เลียนแบบ — ดูส่วน D)
> - ไฟล์ "ต_วอย_าง_*" = ตัวอย่างที่กรอกแล้ว ใช้เทียบผลลัพธ์ (expected output)

---

## 0. กฎการเขียนโค้ด (ทุกเอกสาร)

```java
// ✅ ถูก: เปิด template จริง
Workbook wb = WorkbookFactory.create(
    getClass().getResourceAsStream("/templates/quotation_template.xls"));
Sheet sheet = wb.getSheet("Update");
sheet.getRow(3).getCell(1).setCellValue(issueDate);   // เติมค่าอย่างเดียว

// ❌ ผิด: สร้างใหม่ (นี่คือสาเหตุที่ generate ไม่ตรง template)
Workbook wb = new XSSFWorkbook();          // ห้าม
Sheet sheet = wb.createSheet("Quotation"); // ห้าม
cell.setCellStyle(newStyle);               // ห้ามสร้าง/ทับ style
```

- ใช้ `row.getCell(col)` ของแถวเดิม — **ห้าม createRow ทับแถวที่มีอยู่**, ห้าม `shiftRows` ถ้าไม่จำเป็น (แถวสินค้าใน template มีเผื่อไว้แล้ว)
- ตัวเลขเงิน: เซ็ตเป็น **ตัวเลข** (`setCellValue(double)`) ไม่ใช่ string — เซลล์มี number format ของมันอยู่แล้ว
- วันที่: template ใช้ค่า date จริง (B4 เป็น datetime) → เซ็ตเป็น date และให้ format เดิมของเซลล์แสดงผลเอง (แสดงเป็น พ.ศ. ตาม format ไฟล์)
- หลัง generate: แปลง PDF ผ่าน LibreOffice headless (JODConverter) — **ทดสอบเทียบกับไฟล์ "ต_วอย_าง_*" ทุกช่องต้องตรงตำแหน่ง**

## A. ใบเสนอราคา (QUOTATION) — sheet `Update`

### A1. เงื่อนไขพิเศษ (ตาม requirement)
**ก่อนกด Generate** ต้องมีฟอร์มให้ Sales แก้ค่า 3 ตัวนี้ (มีค่า default):
| # | ฟิลด์ | default | ไปที่ |
|---|---|---|---|
| 1 | `offerDate` — วันที่ได้รับจำนวน | วันนี้ | หมายเหตุข้อ 1 (B24) |
| 2 | `depositPercent` — % มัดจำ | 50 | หมายเหตุข้อ 2 (B26) |
| 3 | `deliveryLeadDays` — ส่งมอบสินค้าภายในกี่วัน | 90 | หมายเหตุข้อ 3 (B28) |
UI: modal/หน้า "ตรวจก่อนออกเอกสาร" → แก้ 3 ค่า → กด Generate

### A2. เซลล์ที่เติม (นอกนั้นห้ามแตะ)
| เซลล์ | ค่า | ที่มา |
|---|---|---|
| `B4` | วันที่ออกเอกสาร (date) | วันที่กด generate |
| `I3` | รหัส salesperson (เช่น `J003`) | user ที่ล็อกอิน (ตาราง signatories/users) |
| `I4` | เลขที่อ้างอิง `QN` + ปีไทย2หลัก + running (เช่น `QN69xxxxx`) | doc number sequence (series QN) |
| `H5` | หน่วยงาน (ถ้ามี) | ticket |
| `H6` | `Sales : {ชื่อ} T.{เบอร์}` เช่น `Sales : Juthatip T.086-3652343` | user Sales ของ ticket |
| `A5`/`B5` | เรียน: `คุณ{ผู้ติดต่อ}   /  {บริษัท}  เลขที่ผู้เสียภาษี : {taxId}` | contacts + customers ของ ticket |
| `B6` | `โทร. {เบอร์}    / E-mail :   {อีเมล}` | contacts |
| `B8` | `Project  : {ชื่อโครงการ}` | projects ของ ticket |
| แถวสินค้า (เริ่ม A10) | ดู A3 | ticket_items ที่อนุมัติ |
| `B24` | `1.จำนวนที่เสนอข้างต้นเป็นจำนวนที่ได้รับมาเมื่อวันที่  {offerDate:d/M/พ.ศ.}` | ฟอร์ม A1 |
| `B26` | `2.บริษัทฯ ขอรับมัดจำ {depositPercent}% เมื่อสั่งซื้อสินค้า ส่วนที่เหลือขอรับ` | ฟอร์ม A1 (B27 บรรทัดต่อ คงเดิม) |
| `B28` | `3.ขณะนี้โรงงานผู้ผลิตประเทศอิตาลีมีสินค้าในสต็อก ระยะเวลานำเข้าประมาณ {deliveryLeadDays} วัน` | ฟอร์ม A1 |
| `I38` | ผลรวม `SUM(เป็นเงิน ทุกรายการ)` | คำนวณ |
| `I39` | `I38 × 0.07` (**VAT บนยอดรวมเต็ม**) | คำนวณ |
| `I40` | `I38 + I39` | คำนวณ |
> B25, B27, B29–B36 (หมายเหตุส่วนที่เหลือ), H39 (0.07) = ของ template อยู่แล้ว **อย่าเขียนทับ**

### A3. รายการสินค้า (แถว 10 เป็นต้นไป — 1 รายการใช้ 3 แถวตามตัวอย่าง)
รูปแบบจากไฟล์ตัวอย่างจริง (รายการที่ 1 = แถว 10–12, รายการที่ 2 = แถว 13–15, ถัดไปเว้นทีละ 3):
| เซลล์ (รายการที่ n เริ่มแถว R=10+3(n-1)) | ค่า |
|---|---|
| `A{R}` | ลำดับ (1, 2, 3…) — template มีเลขอยู่แล้วบางแถว เซ็ตทับได้ |
| `B{R}` | `กระเบื้อง รุ่น {รุ่น} สี {สี} ขนาด {กว้างxยาว} cm.` (ผิว/สเปคเพิ่มตามข้อมูล item) |
| `C{R}` | จำนวน (ตัวเลข) |
| `D{R}` | หน่วย (`แผ่น` / `ตร.ม.`) |
| `E{R}` | ราคา/หน่วย (ตัวเลข — จากราคาที่ CEO อนุมัติ) |
| `F{R}` หรือ `G{R}` | ส่วนลด: % → F, ไม่มีส่วนลด → `G="Net"`, ราคาพิเศษ → `G="พิเศษ"` |
| `H{R}` | คงเหลือ: `Net`→=E, `%`→E−E×F, `พิเศษ`→ราคาพิเศษ |
| `I{R}` | `H × C` |
| `B{R+1}` | `(จำนวนที่ใช้ {จำนวนจริง} แผ่น + เพื่อปัดลงกล่อง =  {จำนวนปัด} แผ่น)` — ใส่เมื่อมีข้อมูลปัดกล่อง |
| `B{R+2}` | `(บรรจุ {pcs_per_box} แผ่น/กล่อง)` — จากฐาน catalog |
> ถ้ารายการเยอะจนเกินโซน (แถว ~10–22): ตัดขึ้นหน้า/ไฟล์ที่สอง หรือแจ้งเตือน — **ห้าม shiftRows ดันหมายเหตุ/สรุปยอด**

## B. ใบแจ้งยอดเงินรับมัดจำ (DEPOSIT_NOTICE) — sheet `Update`
| เซลล์ | ค่า |
|---|---|
| `I7` | วันที่ออก (date) |
| `I8` | เลขที่เอกสาร `GLRD` + ปีไทย + running (series GLRD) |
| `H9` | ต่อท้าย: อ้างอิงใบเสนอราคา/PO เช่น `QN69xxxxx / {เลข PO}` |
| `B7` | ชื่อบริษัทลูกค้า (+สาขา) |
| `B8` | ที่อยู่ลูกค้า — **จากตาราง customers โดยตรง** (template เดิมใช้ VLOOKUP — ห้ามพึ่ง ให้เซ็ตค่าตรง ๆ ทับ) |
| `B11` | `Project : {ชื่อโครงการ}` |
| แถวสินค้า (เริ่มแถว 13, โครงเดียวกับ A3 แต่คอลัมน์ราคาพิเศษ = **J**) | ticket_items |
| `I44` | `ROUND(SUM(เป็นเงิน), 2)` |
| `H45` | depositPercent เป็นทศนิยม (0.5) — จากใบเสนอราคาของ ticket |
| `I45` | `ROUND(I44 × H45, 2)` |
| `I46` | `ROUND(I45 × 0.07, 2)` ← **⚠ VAT คิดจากยอดมัดจำ ไม่ใช่ยอดรวม** |
| `I47` | `I45 + I46` |
> ตรวจกับตัวอย่างจริง: 536,949.6 → มัดจำ 268,474.8 → VAT 18,793.24 → ต้องชำระ **287,268.04**

## C. ใบแจ้งหนี้ส่วนที่เหลือ (REMAINING_INVOICE) — sheet `Update`
| เซลล์ | ค่า |
|---|---|
| `H7` | วันที่ออก |
| `H8` | เลขที่เอกสาร `GLR` + ปีไทย + running (series GLR) |
| `H9` | อ้างอิง PO/ใบเสนอราคา |
| `B7` | ชื่อบริษัทลูกค้า |
| `B8` | ที่อยู่ลูกค้า — เซ็ตตรง ๆ (**template เดิมเป็น `#REF!` — ต้องเขียนทับเสมอ**) |
| `B11` | `Project : {ชื่อโครงการ}` |
| แถวสินค้า (เริ่มแถว 13) | รายการ **ยอดเต็ม** ของสินค้า (โครงเดียวกับ A3) |
| แถวหักมัดจำ (แถวถัดจากรายการสุดท้าย +1 เว้น 1 แถว เช่นแถว 15 ในตัวอย่าง) | `B="หัก  มัดจำ {เลขอ้างอิงมัดจำ}"`, `C=-1`, `E=depositAmount`, `H=depositAmount`, `I=-depositAmount` |
| `I42` | `SUM(เป็นเงิน)` = ยอดเต็ม − มัดจำ (บรรทัดลบรวมในตัว) |
| `I43` | `I42 × 0.07` ← **VAT บนยอดคงเหลือ** |
| `I44` | `I42 + I43` |
> ตรวจกับตัวอย่างจริง: 37,114.18 − 18,557.09 = 18,557.09 → VAT 1,298.9963 → **19,856.0863**
> depositAmount + เลขอ้างอิง ดึงจาก DEPOSIT_NOTICE ที่ออกไปแล้วของ ticket เดียวกัน

## D. Import Request (IMPORT_REQUEST) — template เป็น PDF → สร้าง `.xlsx` เลียนแบบครั้งเดียว
สร้าง `import_request_template.xlsx` ตามภาพ `Template_Import_Request.pdf` (form `F-SM-001 (04)`):
- หัว: `Import Request Form` | `ReF. No.: IR{ปีไทย}{running}` (series IR) | `Brand: {โรงงาน}` | `Request date: {วันที่}`
- ตาราง: `Item | Code | Size | จำนวน | หน่วยนับ | สั่งมาให้ | Requested by | กำหนดวันที่ต้องการของ`
- แถวแรกของตาราง: `Project : {ชื่อโครงการ}` (ตามตัวอย่าง IR69068)
- รายการ: Code=`{รุ่น} {สี} {ผิว}` (เช่น `Lithos Nero Nat`), Size=`{ขนาด} cm`, จำนวน, หน่วยนับ=`pcs`, สั่งมาให้={ลูกค้า/ผู้ติดต่อ}, Requested by={Sales}, กำหนดวันที่ต้องการของ=`Within {วันที่}`
- ท้ายฟอร์ม: กำหนดเรือเข้าโดยประมาณ / Checked / Approve / Request By / วันที่ได้รับมัดจำ — เว้นให้กรอกมือ
- **1 โรงงาน = 1 ใบ**: group `ticket_items` ตาม `factory` (จากฐาน catalog) → ออกหลายใบถ้ามีหลายโรงงาน
- ตัวอย่างเทียบ: `ต_วอย_าง_Import_Request.pdf` (IR69068, Brand Padana, 3 รายการ)

## E. แหล่งข้อมูล (สรุป mapping จาก ticket → เอกสาร)
| ข้อมูล | มาจาก |
|---|---|
| วันที่ | วันที่กด generate |
| เรียน/โทร/อีเมล/เลขภาษี/ที่อยู่ | `customers` + `contacts` ที่ผูกกับ ticket (Sales กรอกไว้ตอนสร้าง ticket) |
| Sales ชื่อ+เบอร์ / รหัส | user Sales เจ้าของ ticket (+ตาราง signatories สำหรับรหัส→ชื่อผู้เซ็น) |
| Project | `projects` ของ ticket |
| รายการสินค้า | `ticket_items` (รุ่น/สี/ผิว/ขนาด/จำนวน/หน่วย) + `pcs_per_box`,`sqm_per_piece` จาก catalog |
| ราคา | ราคาที่ CEO อนุมัติ (`approvedPrice` หรือ `manualPrice`) — **ไม่ใช่ราคา raw** |
| เลขเอกสาร | sequence แยก series: QN / GLRD / GLR / IR + ปีไทย 2 หลัก |
| depositPercent | ค่าจากฟอร์มก่อน generate ใบเสนอราคา → reuse ในใบมัดจำ |

## F. ✅ Acceptance (ทดสอบเทียบไฟล์ตัวอย่างจริง)
1. Generate ใบเสนอราคาจาก ticket ตัวอย่าง → เปิดคู่กับ `ต_วอย_างใบเสนอราคา.xls` **ตำแหน่ง/หน้าตาตรงทุกช่อง** (โลโก้ เส้น merged หมายเหตุ ลายเซ็น F-SM-002 (03) ครบ)
2. ก่อน generate ใบเสนอราคา แก้ค่า 3 ตัว (วันที่ข้อ1 / %มัดจำข้อ2 / ส่งมอบภายในกี่วันข้อ3) ได้ และค่าปรากฏในหมายเหตุ B24/B26/B28 ถูกต้อง
3. ยอดใบเสนอราคา: VAT บนยอดรวมเต็ม (ตัวอย่าง: 129,280.8 → 9,049.656 → 138,330.456)
4. ยอดใบมัดจำ: VAT บนมัดจำ (536,949.6 → 268,474.8 → 18,793.24 → 287,268.04)
5. ยอดใบแจ้งหนี้ส่วนที่เหลือ: มีบรรทัดหักมัดจำติดลบ + VAT บนคงเหลือ (→ 19,856.0863)
6. Import Request แยกใบต่อโรงงาน + เลข IR รัน + มีแถว Project
7. ทุกใบดาวน์โหลดได้ทั้ง .xlsx และ .pdf โดย PDF หน้าตาตรงกับ Excel (ฟอนต์ไทยไม่เพี้ยน — ติดตั้ง TH Sarabun New บนเครื่องที่รัน LibreOffice)
8. ห้ามมีที่ไหนในโค้ดสร้าง Workbook/Sheet ใหม่สำหรับเอกสารพวกนี้ (`new XSSFWorkbook()` = fail review)
